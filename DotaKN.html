Predicting Winning Dota 2 Side With Minimal Features¶

Introduction¶

Dota 2 is a MOBA arena style game where two teams of five fight to destroy the others base. Players play as one of over 100 heroes and work as a team to , collect gold, conquer objectives, and kill the enemy team.

More information on dota 2 and its mechanics can be found here: http://blog.dota2.com

In this tutorial I attempt to determine the winning side of a game as early on as possible in a game. We start with Data available before the game begins and progressively add more revealing information available at later times in a game. This helps to determine the magnitude various actions in-game have on the result of the match.

Data is aquired using the OpenDota API detailed here: https://www.opendota.com

It is clear above that the retrieved data is quite messy and missing many values

The data was fetched in seperate python scripts because the API was unstable with a maximum request limit of 60/min

First we retrieve random public match data followed by a query for more detailed Match information including individiual player performance

def CreateHeroDict():
    heroes = dict()
    for i in range(125):
        heroes[i] = 0
    return heroes

HeroDict¶

This Dict is used to map the hero selections of each team

We later vectorize the dict to use in our classifier

WinningSide = []
X = []
rad_hit = []
op_hit = []
rad_kdas = []
op_kdas = []
radiant_gold_adv = []
for i, row in df.iterrows():
    players = row['players']
    players = ast.literal_eval(players)
    rad_kda = 0
    rad_hits = 0
    rad_gold_adv = 0
    rad_heroes = CreateHeroDict()
    for i,player in enumerate(players[:5]):
        rad_kda += player['kda'] 
        rad_hits += player['last_hits']
        rad_heroes[player['hero_id']] = 1
        rad_gold_adv += player['gold_spent']
        
    op_kda = 0
    op_hits = 0
    op_heroes = CreateHeroDict()
    
    for i,player in enumerate(players[5:]):
        op_kda += player['kda']
        op_hits += player['last_hits']
        op_heroes[player['hero_id']] = 1
        rad_gold_adv -= player['gold_spent']
        
    temp = []
    temp.append(rad_heroes)
    temp.append(op_heroes)
    X.insert(i,temp)
    rad_hit.insert(i,rad_hits)
    op_hit.insert(i,op_hits)
    radiant_gold_adv.insert(i,rad_gold_adv)
    rad_kdas.insert(i,rad_kda/5)
    op_kdas.insert(i,op_kda/5)
      
    
    WinningSide.insert(i,0 if row['radiant_win'] else 1 )

Formatting Data¶

Above we retrieve the data we want to use for classification later

*_heroes: Is the hero dictionary mapping each teams hero choices

*_hits: The number of minion kills totalled for the team. Minions are small enemies that automatically spawn for each team during the game. A majority of the first 15minutes is spent killing these minions

*_kda: The mean Kill/Death/Assist ration for each team

rad_gold_adv: The total gold advantage of the radiant side at the end of the game (negative values imply that the opposing team has the advatage)
